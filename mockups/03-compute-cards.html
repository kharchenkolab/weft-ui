Design notes

  1. Shared badge. Site registered with "shared": true — realizations take build leases and adopt environments others already built (realize.adopted). The badge explains "someone else built this env" moments before they confuse anyone.
  2. Cloud money is always visible. Accrued $ against the cap on the card itself, watchdog state spelled out. budget.watchdog events toast — money never hides in a ticker.
  3. Drift chip. A site.capability_violation anywhere in the app raises "re-probe suggested" here — capabilities are facts with an age, not eternal truths.
  4. Absence with consequence. "Compute internet: no" immediately states what weft does about it (environments arrive packed) — the user learns the realization strategy from the fact.
  5. Honest degradation. QOS limits render as cannot determine — no accounting DB visible, never a blank cell or a fake zero.
  6. Policy editing = re-registration. register_site upserts and re-registration is idempotent (a quick round-trip), so "Edit" re-runs it with the modified config — stated plainly instead of pretending there's a separate settings store.
  7. Audit with actor. The activity slice shows who (agent session vs user button) beside what — both peers land in the same trail.
  8. Start-ETA calculator. Backed by sbatch --test-only via site_load(resources=…); the result is labeled an estimate because that's what it is.
  9. Cheap-to-undo framing. Evict copy states the actual cost of undo (seconds, offline, cache stays warm) so reclaiming disk isn't scary; the expensive variant is a separate, named action.
  10. Consequential GC is confirm-gated and carries weft's warning verbatim — the consent gate treats gc_sweep/gc_packages as account-level tier, same as the agent would face.
