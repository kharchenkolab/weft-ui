Design notes (markers are mockup chrome, not UI)

  1. Outage banner. Honest degradation: the outage is announced with its consequence ("jobs unaffected" — submitted jobs live on the site's scheduler) and the poller's retry schedule, not a mystery red icon. Emitted by site.unreachable; clears on site.reachable. The beamlab row in the table freezes at its last event and is labeled as of 14:28 — stale, not lying; the poller reconciles on reconnect.
  2. Load header. Per-site gauges straight from cached site_load: load fraction for plain hosts, per-partition idle counts + pending backlog for slurm. Cloud sites always show accrued $ against the cap once nonzero — money is never buried in a ticker.
  3. Transfers strip. transfer.progress events render as bytes/rate/ETA bars, cross-linked to the STAGING job they serve. One SSE stream drives this strip, the table diffs, the ticker, and chat digests — the UI never polls.
  4. Keyboard-first. j/k navigate, x select, / search, ⏎ opens detail. Multi-select enables bulk Cancel / Retry failed.
  5. Array row. 2,000 elements collapse to one row driven by coalesced array.progress digests (the UI never consumes per-element events). Segment bar = done/failed/running/queued. Expansion → mockup 02.
  6. Queue REASON chip. Slurm's pending reason verbatim, with a start estimate from site_load(resources=…) (sbatch --test-only) on hover. Estimates are labeled as estimates.
  7. Memoized marker. Identical task_hash → manifest returned instantly; the row says so instead of faking a 0-second run.
  8. "Ask the agent". Opens chat pre-seeded with this job's context — the same cross-link the agent's output gets back into these panels. Peers, both ways.
  9. Error card = renderer, remediations = API calls. The structured error (code · stage · hints · retryable) renders once and is reused verbatim in chat (mockup 05). Fix buttons are the moves the agent would make — sized-up resubmit honors the "never re-size down" playbook rule; the ⌁ caption names the exact call, so the audit trail reads the same either way.
  10. Plan echo. Submit-time promises (staging bytes/ETA, env action, queue estimate) shown against actuals — calibrates trust in the next plan card.
  11. Event ticker. The same bus the agent reads via events_poll; cursor persisted so a UI restart replays, never re-fetches.
